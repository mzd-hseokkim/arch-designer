# AWS Terraform — order-service

Generated by `arch-designer:iac-gen`. Every `resource` / `module` block carries an `ADR-NNNN` comment tracing to `../../docs/adr/`.

## Prereqs
- Terraform ≥ 1.6
- AWS credentials (`AWS_PROFILE` or env vars) for an account where you have admin in `ap-northeast-2`
- ACM certificate issued for your domain (referenced manually in ALB listener — see TODO below)
- ECR repository created and `var.api_image` published

## Apply
```sh
cp backend.tf.example backend.tf  # edit values
terraform init
terraform plan -out tfplan
terraform apply tfplan
```

## What's NOT included (intentional)
- **Secrets values** — DB password is generated by Aurora module (Secrets Manager). FCM server key, OAuth credentials for 채널 API: user uploads via console / `aws secretsmanager`.
- **State backend config** — `backend.tf` is gitignored template; user provisions own S3+DynamoDB.
- **DNS records** — Route 53 zone may be in a separate account.
- **ACM certificate** — must be issued out-of-band; ALB HTTPS listener marked TODO in `main.tf`.
- **IAM users / SSO** — operator access pattern is org-specific.
- **Channel adapter / order-ingest / notification-worker services** — duplicate the `api_service` module call once images exist. Pattern identical per ADR-0003.
